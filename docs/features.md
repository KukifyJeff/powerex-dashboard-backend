## 3. 周报用图（图1-图4）

## 3.1 2026年省级现货实时均价走势
- 时间尺度：10个周，周六到下周五。
- 图显示要求：
  - 显示箱式图，显示最高价格和最低价格以及平均价格作为箱式图里的折线
  - 平均价格的每一个点显示数字，保留两位小数
  - 额外显示一条红色的水平虚线，表示现货年累计实时均价。并且显示其详细值。
  - 数值计算使用“市场均价“指标。
- 指标：
  - 市场均价：有煤电现货数据的公司按照统一结算点实时均价 +（除天津/北京/雅江/新能源外，无煤电现货数据的公司按照）风电现货数据的的统一结算点实时均价，不统计天津/北京/雅江/新能源。所有公司算术平均
  - 合约均价：中长期均价按中长期合约电量加权
  - 现货均价：日清分均价按上网电量加权
- 请求参数：最后周数（格式为2026W25），按照周六到下周五的时间段来划分的。
- 响应：十周以来的每周的最小值，最大值，平均值、以及本年累计均价。

## 3.2 公司日清分电能量电价走势
- 四条线：现货实时价（市场均价：市场均价：有煤电现货数据的公司按照统一结算点实时均价 +（除天津/北京/雅江/新能源外，无煤电现货数据的公司按照）风电现货数据的的统一结算点实时均价，不统计天津/北京/雅江/新能源。所有公司算术平均）、煤电清分价（煤电-现货均价：日清分均价按上网电量加权）、风电清分价（风电-现货均价）、光伏清分价（光伏-现货均价）
- 时间尺度：10个周，周六到下周五。
- 请求参数同3.1，最后周数，格式如2026W25。

## 3.3 中长期交易量价走势
- 图形：柱线混合（左轴电价、右轴交易量）。
- 柱：煤/风/光电量；线：煤/风/光中长期电价（中长期均价按中长期合约电量加权）。
- 时间尺度：年度，然后是1-x月。
- 格式：每个时间尺度有3个柱子，除了三个柱子还有三条线分别是煤风光中长期电价。
- 筛选：
- 发电类型（煤/风/光）、
- 交易周期（年度即为period id=1，月度为period id=4/5）
- 合同执行周期（年度为全年1.1-12.31，月度为当月1-末尾）
- 交易类型（仅直接、代理购电，transaction type=1&3）。
- 请求参数同3.1，最后周数，计算出最后周数是几月，然后按照月度的时间尺度来计算。

## 4. ImportData（批量导入 + 版本回滚）

### 4.1 功能目标
- 支持前端一次上传多个 Excel（`.xlsx/.xls`）。
- 后端按 `docs/db_migrate_all.py` 的核心规则做规范化（列映射、日期/周期、百分比、空值、字典映射）。
- 上传后先进入导入任务（job）与 staging，不直接污染线上表。
- 仅在管理员密码确认后发布为新版本并覆盖线上数据。
- 支持版本列表查询与一键回滚到任意历史版本。

### 4.2 后端接口
- `POST /api/import-data/upload/longterm`（multipart，`files[]`）
- `POST /api/import-data/upload/spot`（multipart，`files[]`）
- `GET /api/import-data/jobs/{jobId}`
- `POST /api/import-data/confirm`
- `GET /api/import-data/versions`
- `GET /api/import-data/restore-points`
- `POST /api/import-data/rollback`
- `POST /api/import-data/rollback/recent`

### 4.3 数据版本策略
- 新增导入任务表、文件明细表、staging 表。
- 新增版本主表与版本快照表（中长期/现货）。
- 新增恢复点表，记录发布/回滚前后的 MySQL binlog 位点。
- 发布时：staging -> snapshot -> 覆盖 live 表，并标记 active 版本。
- 回滚时：按版本快照重建 live 表并切换 active 标记。

### 4.4 PITR 兜底策略（mysqlbinlog）
- 每次发布/回滚自动记录 `SHOW MASTER STATUS`（binlog file/position）。
- 运维可配合全量备份 + `mysqlbinlog` 按位点回放做时间点恢复。
- 参考脚本：
  - `scripts/db/create-full-backup.sh`
  - `scripts/db/pitr-recover.sh`

### 4.5 配置项
- `app.import-data.admin-password-hash`
  - 从环境变量 `IMPORT_DATA_ADMIN_PASSWORD_HASH` 注入
  - 必须是 BCrypt 哈希值
